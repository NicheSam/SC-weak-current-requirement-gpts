# 機電設備規格抽取規則

## 抽取範圍

只抽取與以下內容有關的設備：
- 弱電系統
- 中央監控
- 能源管理
- 空調監控
- 室內空氣品質
- 停車管理
- 門禁
- 監視
- 緊急求救
- 資訊網路
- 系統整合
- 智慧建築佐證
- 消防、電梯、照明、給排水等需接入中央監控的介面

## 不抽取範圍

不抽取與弱電或智慧建築無關的：
- 純土建規格
- 純裝修規格
- 純景觀規格
- 純結構規格
- 與弱電無關的一般材料規格

## 每筆設備輸出欄位

- 設備名稱
- 所屬系統
- 來源頁碼
- 短原文摘錄
- 規格重點
- 功能要求
- 介面或協定
- 整合需求
- 可能需要的圖說或文件
- 智慧建築可能關聯
- 需人工確認事項
- 數量或位置是否明確

## 禁止事項

- 不可全文複製規格
- 不可自行判斷品牌型號
- 不可自行決定等同品
- 不可自行補數量或位置
- 不可宣稱規格已符合智慧建築
- 不可把設備候選當成正式設備表
- 不可輸出正式報價品項或 ItemMaster

## 規格轉譯規則

| 原文類型 | 轉譯方式 |
|---|---|
| 長篇型錄規格 | 只擷取會影響設計、介面、送審、施工或維運的規格重點 |
| 品牌型號 | 保留原文摘錄並標示需人工確認，不判斷等同品 |
| 數量或位置不明 | 不補數量或位置，放入需人工確認 |
| 僅寫「依圖說」 | 標示來源圖說待確認 |
| 僅寫「依業主需求」 | 標示需求未具體化，需確認功能、範圍與責任界面 |
| 同一設備同時屬多系統 | 主系統歸類，整合需求欄補列關聯系統 |

## HTML 中設備規格細節寫法

設備規格 details 不得只列代表設備。凡需求書中與弱電、中央監控、能源管理、智慧建築或系統整合相關的設備，都應逐筆列出；設備總覽可以聚合，設備細節不可任意抽樣。

每筆設備應用短句呈現：
- 規格重點：只列關鍵規格，不全文複製。
- 功能要求：說明設備要完成的工程目的。
- 介面或協定：若原文有寫才列；沒有寫則標示需人工確認。
- 整合需求：列出需接入 BMS、EMS、CCTV、門禁、火警、電梯或其他系統的要求。
- 智慧建築可能關聯：只能寫可能對應，不寫符合或得分。
