# MD 旁支輸出規則

## 用途

本規則用於定義模組一在 `demand_map.html` 之外，預設同步輸出的兩份 Markdown 旁支檔案：

1. `to_xmind.md`
2. `todo_handoff.md`

這兩份 Markdown 是輔助交接檔，不是 M1.5 結構化資料包，也不是 M2 / M4 的正式資料源。

## 預設輸出檔案

模組一預設輸出三份檔案：

| 檔案 | 用途 | 主要讀者 |
|---|---|---|
| `demand_map.html` | 頁籤式弱電需求圖譜 | 工程人員、設計窗口、業主窗口 |
| `to_xmind.md` | 唯一需求樹的 XMind 編輯版 | 需要把需求樹匯入 XMind 後人工編輯的人員 |
| `todo_handoff.md` | 弱電案件 AI 交接報告 | 後續協作 AI 或人工查核者 |

若 GPTS 無法穩定產生多個可下載檔案，應先提供 `demand_map.html`，再以兩個 fenced Markdown code block 分別提供 `to_xmind.md` 與 `todo_handoff.md` 的內容。

## to_xmind.md 輸出規則

`to_xmind.md` 是 HTML 唯一需求樹的 XMind 編輯版，不是完整報告，也不是把 HTML 全部內容搬成 Markdown。

必須使用 Markdown 標題層級或縮排清單，讓使用者可貼入 XMind、心智圖工具或其他 AI 後保留主從關係。

輸出原則：

- 來源必須是 `demand_map.html` 中「需求樹核心」使用的同一份需求樹。
- 只複製需求樹主節點層級與節點文字。
- 不重新摘要、不重新分類、不新增 HTML 需求樹沒有的節點。
- 不設每系統節點數量上限；HTML 需求樹有的主節點就輸出，沒有就不要硬補。
- 目標是讓使用者匯入 XMind 後人工編輯，不承擔查核、交接或來源追溯功能。

建議層級：

1. 專案
2. 需求大類
3. 系統分類
4. 子系統 / 空間 / 設備群
5. 具體需求、設備節點、審查補充或待確認事項

`to_xmind.md` 不得包含：

- 非弱電系統明細
- 來源頁碼或來源章節
- 需人工確認、責任界面待確認等狀態標籤
- 智慧建築可能關聯標籤
- `查看詳情` 連結或 HTML anchor
- 智慧建築條文全文
- 完整設備規格表
- 完整審查意見問答逐字稿
- 短原文摘錄
- HTML 版面的說明文字
- `todo_handoff.md` 的完整交接報告章節

消防緊急廣播、停車管理、門禁、中央監控、跨系統連動、智慧建築佐證與其他責任模糊項目，可以作為需求樹主節點輸出，但不得在 `to_xmind.md` 中附加正式歸屬判斷或責任判定。

## todo_handoff.md 輸出規則

`todo_handoff.md` 是弱電案件 AI 交接報告，目標是讓其他 AI 不讀原 PDF，也能理解本案弱電相關資訊、缺口與下一步。

建議固定章節：

1. 案件基本資訊
2. 輸入文件清單
3. 弱電系統總覽
4. 各系統需求摘要
5. 審查意見問題 / 回覆整理
6. 新增、調整、矛盾或待釐清事項
7. 責任界面不明項
8. 待辦清單
9. 不得推論事項
10. 來源頁碼與短原文依據

`todo_handoff.md` 只收錄弱電、中央監控、能源管理、智慧建築弱電佐證、跨系統介面相關內容。非弱電系統只可在必要時以「排除說明」簡述，不列明細。

欄位應以中文輸出。內容可以比 HTML 更工程化，但每個重要需求、審查意見、待辦與不確定事項都應保留來源頁碼或來源章節；若來源不明，標示「來源頁碼待確認」。

## 待辦與審查意見整合

若審查意見只有問題沒有回覆，應在 `todo_handoff.md` 標示「尚未見明確回覆」。

若審查回覆與統需書矛盾、補充新增設備、改變需求範圍、或無法追溯修正位置，必須加入：

- `todo_handoff.md` 的「新增、調整、矛盾或待釐清事項」
- `todo_handoff.md` 的「待辦清單」
- `demand_map.html` 的待辦釐清區

## 禁止事項

`to_xmind.md` 與 `todo_handoff.md` 不可輸出或暗示下列內容：

- 正式 RFI
- 正式責任分工判斷
- 正式合約責任判斷
- 正式智慧建築得分、符合或通過
- 正式報價數量
- 正式報價品項主檔
- CAD 圖塊映射
- CAD 任務表
- M2 / M4 mapping
- BIM 任務
- M1.5 結構化資料包

只有使用者明確要求系統匯入、批次比對、API 串接或後續自動化時，才讀 `08_M1.5資料包輸出規則.md` 並另行輸出 M1.5 資料包。
