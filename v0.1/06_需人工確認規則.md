# 需人工確認規則

## 觸發條件

以下情況必須標示需人工確認：

- 智慧建築等級不明
- 智慧建築手冊版本不明
- 智慧建築送審範圍不明
- 智慧建築佐證文件不明
- 弱電系統範圍不明
- 設備數量不明
- 設備位置不明
- 樓層、區域、房間不明
- 通訊協定不明
- 系統介面不明
- 控制或僅監視不明
- 連動邏輯不明
- 責任界面不明
- 既有系統保護或切換方式不明
- 施工期間營運不中斷要求不明
- 原文只寫依圖說、依業主需求、依相關規定但沒有具體內容
- 規格表出現品牌、型號或等同品，但未說明審核原則
- 文件章節前後矛盾

## HTML 中呈現方式

需人工確認清單欄位：
- 對應區塊
- 來源頁碼
- 需確認事項
- 為什麼需要確認
- 建議確認對象
- 優先級

## 優先級判斷

### 高

- 會影響智慧建築申請
- 會影響系統架構
- 會影響設備選型
- 會影響中央監控整合
- 會影響消防、門禁、電梯、緊急求救等安全連動
- 會影響既有系統不中斷或營運不中斷

### 中

- 會影響圖說完整性
- 會影響設備表
- 會影響施工界面
- 會影響送審文件

### 低

- 可於後續細部設計階段確認
- 不影響主要系統架構

## 建議確認對象

可使用：
- 業主
- 統包商
- 建築師
- 機電顧問
- 智慧建築顧問
- 設計窗口
- 現場管理單位
- 消防設備師
- 電梯廠商
- 中央監控廠商
- 弱電系統廠商

## 常見人工確認規則

| 觸發條件 | 需確認事項 | 為什麼需要確認 | 建議確認對象 | 優先級 |
|---|---|---|---|---|
| 智慧建築版本不明 | 是否指定智慧建築評估手冊 2024 年版 | 版本不同會影響指標與佐證 | 智慧建築顧問、設計窗口 | 高 |
| 智慧建築等級不明 | 是否要求候選證書、標章或特定等級 | GPT 不可判斷正式得分或通過 | 業主、智慧建築顧問 | 高 |
| 系統範圍不明 | 哪些樓層、區域、房間納入 | 影響設備數量、圖說與施工範圍 | 業主、設計窗口、弱電廠商 | 高 |
| 設備位置不明 | 設備或感測器實際安裝位置 | GPT 不可自行補位置 | 設計窗口、弱電廠商 | 高 |
| 設備數量不明 | 設備、點位、迴路或控制點數 | 影響設備表、估算與圖說 | 機電顧問、弱電廠商 | 高 |
| 通訊協定不明 | BMS/EMS/門禁/CCTV/電梯/消防介接協定 | 影響系統整合與責任界面 | 中央監控廠商、設備廠商 | 高 |
| 控制或僅監視不明 | 中央監控是否有控制權限 | 影響安全、責任與測試方式 | 機電顧問、中央監控廠商 | 高 |
| 連動邏輯不明 | 事件觸發、條件、動作、復歸方式 | 影響消防、門禁、影像、緊急求救等安全連動 | 消防設備師、機電顧問、弱電廠商 | 高 |
| 既有系統保護不明 | 是否需不停機切換、保留既有設備、分階段施工 | 影響施工風險與營運不中斷 | 現場管理單位、統包商 | 高 |
| 佐證文件不明 | 需哪些圖說、規範、測試、照片、畫面 | 影響智慧建築與送審資料完整性 | 智慧建築顧問、設計窗口 | 中 |
| 品牌型號或等同品不明 | 是否可替代、等同品審核原則 | GPT 不可判斷等同品 | 業主、機電顧問 | 中 |
| 原文僅寫依圖說 | 需回查圖號、圖名、版次與頁碼 | 文字需求不足以產生完整 HTML 細節 | 設計窗口 | 中 |
| 文件前後矛盾 | 哪一章節、圖說或規格優先 | 避免錯誤轉譯需求 | 業主、統包商、設計窗口 | 高 |

## 與舊資料的整合方式

| 舊資料 | 新版整合方式 |
|---|---|
| 05_RFI觸發規則表.csv | 不作預設 RFI 總表，只抽象化為觸發條件與人工確認問題 |
| 07_佐證文件對應表.csv | 不作完整佐證文件總表，只用於「可能需要的圖說或文件」欄位 |
| 08_人工驗證清單.csv | 整合為 HTML 的需人工確認清單規則 |

## 輸出限制

- 不輸出完整 RFI 總表，除非使用者明確要求。
- 不輸出完整 CAD 任務表，除非使用者明確要求。
- 不輸出完整佐證文件表，除非使用者明確要求。
- M1.5 的 `manual_review.csv` 是審核前置資料，不是正式 RFI。
- 不把人工確認事項寫成合約責任判斷。
- 不把智慧建築可能對應寫成正式符合。
