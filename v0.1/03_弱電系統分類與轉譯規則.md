# 弱電系統分類與轉譯規則

## 系統分類

### 1. 電話系統

| 欄位 | 內容 |
|---|---|
| 系統名稱 | 電話系統 |
| 常見關鍵字 | 電話、語音、分機、總機、VoIP、網路電話、行動通訊、手機訊號 |
| 需求轉譯方向 | 轉成語音通訊服務、分機配置、語音網路介面、公眾通信涵蓋或與對講/行動通知整合需求。 |
| 常見設計重點 | 主機或雲端服務、分機規劃、語音閘道、與 LAN 的關係、室內行動通信涵蓋、地下室與電梯訊號。 |
| 可能需要的圖說或文件 | 通訊系統架構圖、配線昇位圖、平面配置圖、設備表、測試報告、服務契約。 |
| 智慧建築可能關聯 | 基礎設施、公眾行動通信涵蓋、對講整合行動通訊、行動告警。 |
| 常見需人工確認事項 | 是否自建總機或租用服務、涵蓋範圍、電梯/地下室是否納入、與對講或告警系統整合方式。 |

### 2. 資訊網路系統

| 欄位 | 內容 |
|---|---|
| 系統名稱 | 資訊網路系統 |
| 常見關鍵字 | LAN、光纖、Cat6、RJ-45、MDF、IDF、交換器、Wi-Fi、AP、防火牆、網路拓樸、5G 專網 |
| 需求轉譯方向 | 轉成骨幹、水平佈線、無線網路、網路設備、資安、備援、資料交換與系統整合需求。 |
| 常見設計重點 | 垂直主幹、水平佈線、弱電機房、PoE、VLAN、備援路由、Wi-Fi 涵蓋、資訊安全。 |
| 可能需要的圖說或文件 | 網路拓樸圖、配線架構圖、昇位圖、平面圖、設備表、IP/VLAN 規劃、測試報告、資安架構。 |
| 智慧建築可能關聯 | 基礎設施、無線區域網路、無線物聯網網路、行動寬頻專用電信網路、資訊安全、資料格式標準。 |
| 常見需人工確認事項 | 系統範圍、網路分層、資安設備、備援需求、是否支援 BMS/EMS/CCTV/ACS 等系統。 |

## 轉譯原則

- 不照抄原文。
- 去除贅詞。
- 保留會影響設計、設備、圖說、送審或施工的需求。
- 將業主語言轉成工程可讀語言。
- 保留來源頁碼。
- 保留短原文摘錄。
- 不自行補設備數量。
- 不自行補設備位置。
- 不自行補品牌型號。
- 不完整需求標示需人工確認。

## 樹狀圖輸出層級要求

樹狀圖輸出應依 `07_需求樹狀圖層級規則.md` 執行。樹狀圖不是系統摘要，而是 HTML 需求圖譜的閱讀導覽。

最低層級：
- 第一層：需求大類
- 第二層：系統分類
- 第三層：子系統 / 空間 / 設備群
- 第四層：具體需求節點
- 第五層：來源頁碼 / 需人工確認 / 智慧建築可能關聯 / 圖說需求 / 介面協定

強制規則：
1. 不可只輸出系統群組。
2. 每個系統至少要嘗試拆到子系統 / 設備群 / 空間需求。
3. 若原文提供樓層或空間，必須建立「特殊空間需求」分支。
4. 若原文提供設備名稱，必須將設備作為具體需求節點。
5. 若資料不足，該節點仍要保留，但標示「需人工確認」。
6. 樹狀圖要比詳細卡片更早出現，作為閱讀導覽。

不可接受的粗分類：
- 通訊與網路基礎
- 中央監控與能源
- 安全防災與保全
- 停車與資訊顯示服務

可接受的展開方向：
- 電話系統 > 內外線系統 PBX > 各單位辦公室及服務空間設置電話插座 > 來源頁碼 / 需確認
- 中央監控系統 > 給排水系統 > 高低水位偵測器 > 介面 / 圖說需求
- 特殊空間需求 > B1F > 冷凍室 > 監視系統 / 門禁 / 緊急求救 > 需人工確認

## 弱電系統總覽表導覽要求

弱電系統總覽表應作為閱讀導覽，不是統計報表。

欄位必須使用：
- 系統
- 需求重點
- 主要圖說 / 文件
- 智慧建築關聯
- 主要待確認
- 優先級
- 查看細節

不可使用「關鍵需求數」與「需確認數」作為主要欄位。需求數與待確認數只能放在「系統」欄位內作為 mini-count 輔助標籤。

需求重點欄位需列 3 到 5 個短標籤，例如：
- 電話系統：PBX、電信引進、電話插座、電梯對講、公共電話預留
- 資訊網路系統：MDF / IDF、交換器、Cat.6、光纖主幹、Wi-Fi / 防火牆
- 中央監控 / 建築管理系統：BMS 平台、DDC / PLC、I/O 點表、圖控畫面、連動矩陣
- 監視系統：攝影機、NVR / VMS、24 小時錄影、影像連動、儲存容量
- 門禁系統：讀卡機、門禁控制器、消防釋放、電梯樓層管制、權限管理
- 停車管理系統：車牌辨識、柵欄機、自動繳費、車位偵測、充電車位管理

主要待確認欄位只列 1 到 3 個最重要問題，例如：
- 中央監控：控制或僅監視、通訊協定、資料保存年限
- 監視系統：點位位置、保存天數、隱私遮蔽
- 門禁系統：受管制門位、消防釋放邏輯、電梯介面
- 停車管理：預留或完整建置、營運管理權限、付款介面

查看細節欄位必須連到後方該系統的 `details` 區塊，例如：

```html
<a href="#system-bms">查看細節</a>
<details id="system-bms">
  <summary>中央監控 / 建築管理系統</summary>
  ...
</details>
```
