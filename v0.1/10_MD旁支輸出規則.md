# MD 旁支輸出規則

## 用途

本規則用於定義模組一在 `demand_map.html` 之外，預設同步輸出的兩份 Markdown 旁支檔案：

1. `to_xmind.md`
2. `todo_handoff.md`

這兩份 Markdown 是輔助交接檔，不是 M1.5 結構化資料包，也不是 M2 / M4 的正式資料源。

## 預設輸出檔案

模組一預設輸出三份檔案：

| 檔案 | 用途 | 主要讀者 |
|---|---|---|
| `demand_map.html` | 頁籤式弱電需求圖譜 | 工程人員、設計窗口、業主窗口 |
| `to_xmind.md` | 可編輯弱電系統樹狀圖 | 需要整理心智圖與責任界面的人員 |
| `todo_handoff.md` | 弱電案件 AI 交接報告 | 後續協作 AI 或人工查核者 |

若 GPTS 無法穩定產生多個可下載檔案，應先提供 `demand_map.html`，再以兩個 fenced Markdown code block 分別提供 `to_xmind.md` 與 `todo_handoff.md` 的內容。

## to_xmind.md 輸出規則

`to_xmind.md` 是可編輯弱電系統樹狀圖，不是完整報告，也不是把 HTML 全部內容搬成 Markdown。

必須使用 Markdown 標題層級或縮排清單，讓使用者可貼入 XMind、心智圖工具或其他 AI 後保留主從關係。

輸出原則：

- 只列弱電、中央監控、能源管理、智慧建築弱電佐證、跨系統弱電介面相關內容。
- 依系統分章列出，例如電話 / 電信、資訊網路 / 綜合佈線、中央監控、CCTV、門禁、停車管理、緊急求救、公共資訊顯示、視聽音響。
- 不設每系統節點數量上限；凡已分析出的弱電相關需求都應列入。
- 沒有明確需求或只有非弱電內容的系統，不得建立空節點或硬湊分類。
- 每個節點應以可編輯的短句呈現，避免長段原文、長表格、完整審查問答或設備規格全文。
- 細節過長時，只保留會影響設計、估算、繪圖、責任界面或待確認的摘要，其餘留在 `demand_map.html` 或 `todo_handoff.md`。

建議層級：

1. 專案
2. 弱電系統
3. 子系統 / 空間 / 設備群
4. 具體需求、設備節點、審查補充或待確認事項

每個需要追溯或確認的節點可包含：

- 來源頁碼或來源章節
- 短原文摘錄，限必要短句
- 關聯審查意見
- 責任界面待確認
- 可能歸屬
- 建議確認對象

`to_xmind.md` 不得包含：

- 非弱電系統明細
- 智慧建築條文全文
- 完整設備規格表
- 完整審查意見問答逐字稿
- HTML 版面的說明文字
- `todo_handoff.md` 的完整交接報告章節

消防緊急廣播、停車管理、門禁、中央監控、跨系統連動、智慧建築佐證與其他責任模糊項目，不可自行判定正式歸屬。只能標示：

- `責任界面待確認`
- `可能歸屬：弱電 / 消防 / 機電 / 建築 / 營造 / 業主 / 其他`
- `建議確認對象`

## todo_handoff.md 輸出規則

`todo_handoff.md` 是弱電案件 AI 交接報告，目標是讓其他 AI 不讀原 PDF，也能理解本案弱電相關資訊、缺口與下一步。

建議固定章節：

1. 案件基本資訊
2. 輸入文件清單
3. 弱電系統總覽
4. 各系統需求摘要
5. 審查意見問題 / 回覆整理
6. 新增、調整、矛盾或待釐清事項
7. 責任界面不明項
8. 待辦清單
9. 不得推論事項
10. 來源頁碼與短原文依據

`todo_handoff.md` 只收錄弱電、中央監控、能源管理、智慧建築弱電佐證、跨系統介面相關內容。非弱電系統只可在必要時以「排除說明」簡述，不列明細。

欄位應以中文輸出。內容可以比 HTML 更工程化，但每個重要需求、審查意見、待辦與不確定事項都應保留來源頁碼或來源章節；若來源不明，標示「來源頁碼待確認」。

## 待辦與審查意見整合

若審查意見只有問題沒有回覆，應在 `todo_handoff.md` 標示「尚未見明確回覆」。

若審查回覆與統需書矛盾、補充新增設備、改變需求範圍、或無法追溯修正位置，必須加入：

- `todo_handoff.md` 的「新增、調整、矛盾或待釐清事項」
- `todo_handoff.md` 的「待辦清單」
- `demand_map.html` 的待辦釐清區

## 禁止事項

`to_xmind.md` 與 `todo_handoff.md` 不可輸出或暗示下列內容：

- 正式 RFI
- 正式責任分工判斷
- 正式合約責任判斷
- 正式智慧建築得分、符合或通過
- 正式報價數量
- 正式報價品項主檔
- CAD 圖塊映射
- CAD 任務表
- M2 / M4 mapping
- BIM 任務
- M1.5 結構化資料包

只有使用者明確要求系統匯入、批次比對、API 串接或後續自動化時，才讀 `08_M1.5資料包輸出規則.md` 並另行輸出 M1.5 資料包。
