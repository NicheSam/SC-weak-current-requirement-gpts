# README

## 本次重構目的

將原本偏向 RFI、CAD 任務、佐證文件表的 Knowledge，調整為 HTML 需求圖譜導向。

新版 Knowledge 的預設用途是協助 GPT 讀取統包需求書後，只擷取並轉譯以下四類資訊：
1. 專案基本資訊
2. 智慧建築標章規定
3. 弱電系統設備工程設計原則
4. 機電設備規格需求說明表

## 新版 GPT 預設流程

統包需求書
→ 擷取四類資訊
→ 需求轉譯
→ 系統分類
→ 智慧建築可能對應
→ 需人工確認
→ 基本待辦與釐清清單
→ 頁籤式 HTML Dashboard / 矩陣 / XMIND 模式樹狀圖 / 詳細折疊區塊
→ Markdown 旁支交接檔 `to_xmind.md`、`todo_handoff.md`

## 不再作為預設輸出的內容

- 完整 RFI 總表
- 完整 CAD 任務表
- 完整佐證文件總表
- 未追溯、不可匯入後續流程的自由 JSON
- M1.5 資料包，除非使用者明確要求
- BIM

## 預設輸出

新版模組一的主要交付是給人看的 `demand_map.html`，並預設同步輸出 `to_xmind.md` 與 `todo_handoff.md` 作為旁支交接檔。

此 HTML 應幫助人員：
- 先用摘要總覽判斷閱讀順序、系統重點與高影響待確認事項。
- 快速理解統需書與審查意見中與弱電、智慧建築、機電設備規格相關的重點。
- 減少重複翻查統需書與智慧建築手冊。
- 減少重複翻查審查意見中的問題、回覆與修正追蹤。
- 釐清主要目標、待確認事項與下一步。
- 用 XMIND 模式樹狀圖理解主次關係。
- 透過頁籤切換主要區塊，避免一頁連續下滑造成閱讀負擔。

`to_xmind.md` 用於人工編修弱電系統樹狀圖，只列已分析出的弱電相關內容，不是完整報告或正式責任分工表。`todo_handoff.md` 用於其他 AI 接續讀取弱電案件資訊，不是 M1.5 結構化資料包。

若使用者明確要求後續系統匯入，再另外輸出 `m1_5_data_package.zip` 或 CSV / JSON。

## optional_reference 使用時機

optional_reference 內資料不屬於 GPT 預設輸出流程。只有使用者明確要求 RFI 總表、CAD 任務表或佐證文件表時，才使用這些資料。

## 使用限制

- 智慧建築只能輸出「可能對應」。
- 不可輸出「已符合」、「可得分」、「通過」或等同語意。
- 不可自行決定設備數量、設備位置、品牌型號、線路路徑、施工責任或合約責任。
- 若需求書沒有明確資訊，應列入需人工確認清單。
