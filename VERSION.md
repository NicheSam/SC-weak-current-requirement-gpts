# 版本紀錄

## v0.2

封裝日期：2026-07-02

狀態：目前版本。將模組一由 Dashboard-first 調整為 Tree-first。

包含能力：

- 預設開啟 `需求樹核心`。
- 先產生唯一需求樹，再派生 HTML 需求樹與 `to_xmind.md`。
- HTML 分成兩套並行架構：需求樹及其延伸應用、詳細資訊查詢。
- 詳細資訊查詢收錄來源頁碼、短摘錄、審查意見、設備規格與待辦釐清。
- `to_xmind.md` 只複製需求樹主節點層級與節點文字，不含來源頁碼、狀態標籤、審查意見或短摘錄。

已知限制：

- 仍是 GPTS + HTML + Markdown 架構，沒有外部後端或資料庫。
- 仍不與 CAD、QTO、報價單或 M2 / M4 mapping 強連動。
- 需求樹節點與詳細資訊 anchor 的一致性需靠 GPTS 規則約束，後續若要更穩定可考慮外部產生器。

## v0.1

封裝日期：2026-07-02

狀態：封裝現有模組一 GPTS Knowledge 與提示詞，作為 v0.1 基準版本。

包含能力：

- 統包需求書弱電需求轉譯。
- 審查意見弱電相關問題 / 回覆整合。
- 頁籤式 `demand_map.html`。
- 分析結果摘要總覽。
- 弱電系統樹狀圖與 `to_xmind.md`。
- AI 交接用 `todo_handoff.md`。
- M1.5 選用資料包規則，但非預設主流程。

已知問題：

- 使用者回饋摘要後資訊仍偏多。
- HTML-first 結構仍偏完整報告導向。
- XMind 與待辦輸出容易與 HTML 詳細內容重複。
- GPTS Knowledge 上傳可能受檔名、檔案數或平台狀態影響。

下一版方向：

- `v0.2` 將重新設計核心機制，從閱讀場景回推摘要、篩選、分層與待辦輸出。
